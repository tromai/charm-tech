# Security policy

## Supported versions

This repository contains internal Charm Tech team documentation, scripts, and
agent skills. It is not a shipped product and carries no security-maintenance
commitment or supported-version matrix. Reported vulnerabilities will be
investigated promptly, but there are no guaranteed remediation timelines.

## Reporting a vulnerability

The easiest way to report a security issue is through
[GitHub's security advisory for this project](https://github.com/canonical/charm-tech/security/advisories/new).
See [Privately reporting a security vulnerability](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing/privately-reporting-a-security-vulnerability)
for instructions.

You may also send email to security@ubuntu.com. If you want to encrypt your email,
follow [Canonical's reporting instructions](https://ubuntu.com/security/disclosure-policy#contact-us).

The [Ubuntu Security disclosure and embargo policy](https://ubuntu.com/security/disclosure-policy)
contains more information about what you can expect when you contact us, and what
we expect from you.
